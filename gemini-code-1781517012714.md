## 1. Case Study: ACP-CAD-RUNTIME Analysis
* **Задача:** Декомпозиция механического узла (редуктор).
* **Применение примитивов:**
    * Входящая энергия -> **ДЖЕРЕЛО (SOURCE)**.
    * Передача вращающего момента -> **ПОТІК (FLOW)**[cite: 1].
    * Изменение передаточного числа -> **ПЕРЕТВОРЕННЯ (TRANSFORM)**[cite: 1].
    * Ограничения по нагрузке -> **ОБМЕЖЕННЯ (CONSTRAINT)**[cite: 1].
* **Результат:** Точная инженерная модель без семантического дрейфа, соответствующая протоколу ACP-CAD-RUNTIME[cite: 1].

## 2. Case Study: Infrastructure Diagnostics (Utility Service)
* **Задача:** Оптимизация потока ресурсов в системе водоснабжения.
* **Применение примитивов:**
    * Резервуар хранения -> **НАКОПИЧУВАЧ (STORAGE)**[cite: 1].
    * Регулирующий клапан -> **КЕРУВАННЯ (CONTROL)**[cite: 1].
    * Потери на трение -> **РОСІЯННЯ (DISSIPATE)**[cite: 1].
* **Результат:** Модель, минимизирующая потери (TDI < 0.2), применимая для оперативного управления ресурсами[cite: 1].

## 3. Case Study: Liturgical Phonetics Engine
* **Задача:** Генерация псалмодической последовательности.
* **Применение примитивов:**
    * Акустический сигнал -> **ПОТІК (FLOW)**[cite: 1].
    * Артикуляционная точка -> **СПРЯЖЕННЯ (INTERFACE)**[cite: 1].
* **Результат:** Фонетически точная реализация с использованием Sacred Pentaglot core, поддерживающая целостность культурного слоя[cite: 1].
