# CHINESE CORE PASSPORT

## 1. Kernel Identification
* **Kernel ID:** `Chinese-Core`
* **Version:** 3.0.0
* **Status:** Level 3 (Reference Implementation)
* **Architecture:** AAM-V1 / SOL-ECC-1

## 2. Purpose
Эталонная реализация языкового ядра для китайского языка. Обеспечивает детерминированный анализ китайского текста (упрощенные иероглифы) с использованием 9 осей анализа, исключая семантический дрейф и обеспечивая интеграцию с доменными пакетами.

## 3. Language Profile
* **Linguistic Scope:** Изолирующий, тональный язык (путунхуа).
* **Script System:** Упрощенные иероглифы (Simplified Chinese).
* **Tone/Prosody Model:** 4 тона + нейтральный тон.

## 4. Analysis Axes
Ядро реализует 9 осей анализа:

| Axis | Name | Description | Function |
| :--- | :--- | :--- | :--- |
| **Axis-1** | Phonetic | Фонетика, просодия, тональность | Стабилизация звуковой формы |
| **Axis-2** | Radical | Радикалы (部首), графемика | Выделение атомарных единиц |
| **Axis-3** | Syntactic | Порядок слов (SVO), частицы | Определение структуры связи |
| **Axis-4** | Semantic | Смыслы, многозначность | Извлечение смысла |
| **Axis-5** | Pragmatic | Интенции, вежливость, контекст | Анализ цели высказывания |
| **Axis-6** | Cultural | Идиомы, табу, история | Коррекция семантического дрейфа |
| **Axis-7** | Cognitive | Ментальные модели, предположения | Анализ глубинных моделей |
| **Axis-8** | SOL Mapping | Привязка к [SOURCE], [CONSTRAINT] | Преобразование в SOL |
| **Axis-9** | Domain Interface | Подключение доменов (L3) | Интеграция с прикладным API |

## 5. Tone System
* **Тон 1 (阴平, yīnpíng):** Высокий ровный (стабильность).
* **Тон 2 (阳平, yángpíng):** Восходящий (инициация [SOURCE]).
* **Тон 3 (上声, shǎngshēng):** Низкий ломаный (накопление [STORAGE]).
* **Тон 4 (去声, qùshēng):** Нисходящий (завершение [DISSIPATE]).
* **Нейтральный (轻声, qīngshēng):** Слабый (акцент на прагматике).

## 6. Radical System
Система из 214 радикалов (Канси) используется для классификации иероглифов. Радикалы выступают в качестве первичных дескрипторов семантики, обеспечивая связь между визуальной формой и базовым SOL-примитивом.

## 7. Semantic Stabilization Model (720°)
Механизм стабилизации:
1. **Pass-1 (Syntax):** Анализ иероглифической структуры и частиц.
2. **Pass-2 (Semantics):** Маппинг иероглифа на SOL-примитив.
3. **Pass-3 (Causal Consistency):** Verification of semantic dependency chains (с возможностью использования культурного инструментария 五行).
4. **Pass-4 (Consistency Check):** Верификация TDI (Tensor Divergence Index).

## 8. SOL Integration
Интеграция с примитивами:
* **[SOURCE]:** Инициация (напр., иероглиф 源).
* **[CONSTRAINT]:** Предел/граница (напр., иероглиф 限).
* **[FLOW]:** Движение (напр., иероглиф 流).
* **[TRANSFORM]:** Изменение формы (напр., иероглиф 转).
* **[VERIFY]:** Верификация (напр., иероглиф 证).
* **[DISSIPATE]:** Рассеивание (напр., иероглиф 散).

## 9. Domain Compatibility
* **Medicine:** Полная поддержка TCM (традиционная медицина), биомеханика.
* **Finance:** Стандарты ASBE, терминология банковского сектора.
* **Toponymy:** Система адресации и административного деления КНР.
* **Hierarchy:** Система официальных и социальных обращений.

## 10. Reference Status
Chinese Core является эталонной реализацией (Level 3):
* Designed for Benchmark Suite compliance.
* Semantic drift mitigation mechanisms implemented.

## 11. Repository Integration
Chinese Core реализуется в директории `language/chinese/` и содержит файлы:
* `passport.md` — Паспорт ядра.
* `benchmarks.md` — Набор тестов соответствия.
* `failure_modes.md` — Документация по ограничениям.
* `examples/` — Каталог эталонных примеров анализа.

## 12. Reference Implementation Scope
Chinese Core — первая эталонная реализация архитектуры AAM-Language-Kernels. Назначение:
* Проверка архитектуры.
* Тестирование шаблона.
* Основа для: English Core, German Core, Russian Core, Ukrainian Core, Sanskrit Core.

## 13. Dependencies
* **L0:** AAM Meta Core.
* **L1:** ECC-1 Kernel.

## 14. Versioning Policy
* **Major:** Изменение набора осей или структуры иероглифического маппинга.
* **Minor:** Добавление новых доменных пакетов (Domain Packs).
* **Patch:** Корректировка словаря Failure Modes.
