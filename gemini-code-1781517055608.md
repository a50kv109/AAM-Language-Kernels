## 1. Technical Divergence Index (TDI) Definitions
TDI — это комплексный показатель, отражающий «расстояние» между Engineering Intent и текущей лингвистической материализацией.

$$TDI = w_1 \cdot \Delta_{sem} + w_2 \cdot \Delta_{str}$$

Где:
*   $\Delta_{sem}$ — семантическая дистанция (расхождение в SOL-примитивах)[cite: 1].
*   $\Delta_{str}$ — структурный шум (несоответствие грамматических конструкций иерархии AAM-V1)[cite: 1].
*   $w_1, w_2$ — весовые коэффициенты (по умолчанию 0.5/0.5)[cite: 1].

## 2. Performance Zones
*   **Green Zone (TDI < 0.30):** Штатная работа. Система стабильна, семантический дрейф минимален[cite: 1].
*   **Yellow Zone (0.30 ≤ TDI < 0.45):** Требуется аудит. Необходимо проверить `failure_modes.md` и выполнить ре-маппинг терминов[cite: 1].
*   **Red Zone (TDI ≥ 0.45):** Критический сбой. Активация «Алёшиных Ворот» (Alyosha Gate), полная остановка текущей ветки логики и возврат к примитивам SOL[cite: 1].

## 3. Verification Protocol
1.  **Unit Test:** Анализ каждого инженерного термина на соответствие `axes.md`[cite: 1].
2.  **Integration Test:** Проверка цепочки SOL-примитивов на логическую связность (от SOURCE до DISSIPATE)[cite: 1].
3.  **TDI Calculation:** Автоматический расчет индекса для каждого завершенного анализа[cite: 1].

## 4. Benchmark Targets
*   **Target TDI:** < 0.25 для всех инженерных документов[cite: 1].
*   **Drift Resolution:** 100% известных инцидентов из `failure_modes.md` должны иметь закрытый статус в верификационном журнале[cite: 1].
