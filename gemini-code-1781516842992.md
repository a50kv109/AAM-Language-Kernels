## 1. Mapping Protocol
Данный документ является обязательным справочником для всех инженерных операций. Примитив SOL является первичным (инвариантным), украинский термин — вторичным (инструментальным)[cite: 1]. Прямой перевод запрещен — допустимо только отображение (mapping) согласно заданным осям.

## 2. SOL-UA Semantic Axes

| SOL-Примитив | Украинский эквивалент | Инженерное определение (UA) |
| :--- | :--- | :--- |
| **SOURCE** | ДЖЕРЕЛО | Исходная точка потока энергии, данных или материи. |
| **FLOW** | ПОТІК | Направленное перемещение ресурса в системе. |
| **STORAGE** | НАКОПИЧУВАЧ | Зона временной или постоянной аккумуляции ресурса. |
| **CONSTRAINT** | ОБМЕЖЕННЯ | Правило, граница или физический допуск системы. |
| **INTERFACE** | СПРЯЖЕННЯ | Точка сопряжения функциональных узлов. |
| **TRANSFORM** | ПЕРЕТВОРЕННЯ | Изменение формы, состояния или параметров ресурса. |
| **DISSIPATE** | РОСІЯННЯ | Потери энергии, демпфирование или рассеивание смысла. |
| **CONTROL** | КЕРУВАННЯ | Механизм регуляции и обратной связи. |

## 3. Usage Rules
1. **Priority:** При возникновении семантического дрейфа приоритет отдается определению примитива SOL[cite: 1].
2. **Strictness:** Запрещено использовать термины-синонимы, не зафиксированные в данной таблице, без предварительного обновления `axes.md` и согласования с Meta-Core.
3. **Drift Handling:** Любое отклонение от использования фиксированных терминов классифицируется как структурный шум и подлежит коррекции в `failure_modes.md`[cite: 1].
