# Human Interface: Semantic Mapping Matrix

## 1. Mapping Matrix (Subjective -> Objective)

| Human Phrase (Subjective) | Semantic Intent | SOL Primitive | Engineering State |
| :--- | :--- | :--- | :--- |
| "I feel heavy/no strength" | Resource Depletion | [COST] | [DEPLETED_RESERVE] |
| "Something is blocking me" | Stability/Constraint | [CONSTRAINT] | [FLOW_RESTRICTED] |
| "I am afraid to move" | Control/Agency | [CONTROL] | [INHIBITED_STATE] |
| "Everything is messy/fuzzy" | Data Quality | [SIGNAL] | [SIGNAL_NOISE] |
| "At a crossroads" | Decision/Choice | [TRANSFORM] | [PATH_DIVERGENCE] |
| "Everything is going fast" | Temporal Window | [TIME_WINDOW] | [HIGH_VELOCITY] |

## 2. Agency Pattern Rules
* **Rule A**: Never map emotional input to clinical labels. Map to system signal flags.
* **Rule B**: If "Human Phrase" contains multiple intent-types, prioritize the [COST] primitive (depletion is always the primary diagnostic lead).
* **Rule C**: Ambiguity in human speech is treated as [VARIABILITY] (the system must request a constraint-tightening question).

## 3. Implementation
The Mapping Matrix must be accessed by L2 Language Kernels before data is sent to the L3 Domain Packs.
