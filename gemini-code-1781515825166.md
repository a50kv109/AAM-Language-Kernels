# SOL PRIMITIVES: TRILINGUAL DICTIONARY (ENG/CHN/RUS)

## 1. Purpose
Стандартизированный словарь примитивов Semantic Object Language (SOL). Обеспечивает семантическую инвариантность при работе архитектуры AAM-V1 во всех языковых ядрах.

## 2. Dictionary Matrix

| SOL Primitive | English Translation | Chinese Translation | Russian Translation | Engineering Definition |
| :--- | :--- | :--- | :--- | :--- |
| **[SOURCE]** | Initiation Point | 源 / 起点 | Инициация / Источник | Точка входа ресурса или сигнала |
| **[CONSTRAINT]** | Barrier / Limit | 限制 / 界限 | Ограничение / Предел | Граница пространства решений |
| **[FLOW]** | Movement / Current | 流 / 动态 | Поток / Движение | Перенос данных, энергии или массы |
| **[TRANSFORM]** | Conversion / Process | 转换 / 变化 | Преобразование | Изменение параметров состояния |
| **[VERIFY]** | Comparison / Logic | 验证 / 核实 | Верификация / Сравнение | Сверка актуального и эталонного |
| **[DISSIPATE]** | Termination / Exit | 散 / 耗散 | Рассеивание / Завершение | Выход ресурса или отказ системы |

## 3. Implementation Rules
1. **Case Sensitivity:** SOL-примитивы пишутся строго в верхнем регистре в квадратных скобках.
2. **Usage:** При анализе любого текста (Axis-8) лингвистическая единица обязана быть маппирована на один из вышеуказанных примитивов.
3. **Reference:** Для Chinese Core (Level 3) иероглифический эквивалент является жестко закрепленным (см. колонку "Chinese Translation").

## 4. Stability Notes
* [SOURCE] и [DISSIPATE] задают жизненный цикл процесса (Lifecycle Start/End).
* [CONSTRAINT] и [VERIFY] задают логическую устойчивость (Logic Stability).
* [FLOW] и [TRANSFORM] задают динамическую работу системы (System Workload).

## 5. Maintenance
* При добавлении новых языковых ядер (напр., German, Sanskrit) словарь должен быть дополнен соответствующими эквивалентами без изменения инженерного смысла примитивов.
