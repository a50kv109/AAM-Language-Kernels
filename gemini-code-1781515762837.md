# Medical Domain: English-to-SOL Mapping Matrix

## 1. Overview
This module acts as the L3 Domain Pack for Medical/Biomechanical analysis. It bridges subjective English patient descriptions with the deterministic AAM primitive system.

## 2. Mapping Matrix (Subjective to Objective)

| Symptom (English) | SOL Mapping | Engineering State | Diagnostic Lead |
| :--- | :--- | :--- | :--- |
| "Heavy limbs" / "No strength" | [COST] | DEPLETED_RESERVE | Metabolic flux analysis |
| "Stiffness in shoulder" | [CONSTRAINT] | FLOW_RESTRICTED | Kinematic chain audit |
| "Afraid to move" | [CONTROL] | INHIBITED_STATE | Proprioceptive feedback check |
| "Messy/fuzzy feeling" | [SIGNAL] | SIGNAL_NOISE | Neuro-sensory integration audit |
| "Difficult choice" | [TRANSFORM] | PATH_DIVERGENCE | Decision-load capacity check |
| "Fast pace/rushing" | [TIME_WINDOW] | HIGH_VELOCITY | Temporal pacing threshold audit |

## 3. Ambiguity Resolution (Failure Mode Prevention)

* **Case: Stiffness**
    * *General English*: Mental inflexibility or "tightness".
    * *Biomechanical Domain*: Glenohumeral or joint ROM (Range of Motion) restriction.
    * **Stabilization**: Map to `[CONSTRAINT:stability]`. Prohibition: Do not map to psychological states.

* **Case: Powerless**
    * *General English*: Emotional frustration.
    * *Biomechanical Domain*: Loss of motor unit recruitment or proprioceptive feedback.
    * **Stabilization**: Map to `[CONTROL:inhibited]` or `[COST:depletion]`. Prohibition: Do not map to emotional weakness.

## 4. Domain Logic (Axis-9 Workflow)
1. **INPUT**: Token parsing via `L2_language_kernel.py`.
2. **CONTEXT_CHECK**: Axis-7 (Cognitive) verification: Is the input a symptom description or a subjective metaphor?
3. **MAPPING**: Assign [COST] (adaptation price) and [FATIGUE] (resource depletion).
4. **OUTPUT**: Diagnostic output structured as:
   `[FINDING: Biomechanical state] + [RISK: Divergence from norm] + [RECOMMENDATION: Restoration action]`
