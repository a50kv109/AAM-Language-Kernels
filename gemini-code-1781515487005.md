# CHINESE CORE: ECC-1 ENGINE SPECIFICATIONS

## 1. Scope
Спецификация интеграции L1 (ECC-1) с китайским языковым ядром. Описывает, как именно происходят декомпозиция и стабилизация китайских иероглифических конструкций.

## 2. ECC-1 Core Operations (Chinese Implementation)

### DECOMPOSITION
* **Mechanism:** Анализ иероглифа на [RADICAL] + [PHONETIC_COMPONENT].
* **Rule:** Любая иероглифическая единица раскладывается до уровня 214 радикалов [SOURCE].
* **Output:** Древовидная структура иерархии радикалов.

### MAPPING
* **Mechanism:** Привязка иероглифического значения к SOL-примитиву (L0).
* **Rule:** Иероглиф [动] (движение/действие) маппится на [FLOW]. Иероглиф [定] (фиксировать) маппинг на [CONSTRAINT].

### VERIFICATION
* **Mechanism:** Проверка семантической цепочки через 720°-цикл.
* **Rule:** Если TDI > 1.0, система ищет идиоматический контекст (Axis-6).

### STABILIZATION
* **Mechanism:** Проверка просодики (тональности) и синтаксических частиц (Axis-1, Axis-3).
* **Rule:** Окончательный вывод выдается только после подтверждения тонального контекста (в инженерном диалоге часто используются омофоны).

## 3. Stabilization Cycle (Execution Flow)
Процесс анализа (720°):
1. **PASS-1 (Syntax):** Разбор предложений по частицам (的, 地, 得, 了).
2. **PASS-2 (Semantics):** Перевод в SOL-примитивы.
3. **PASS-3 (Causal Consistency):** Концептуальная проверка цепочки зависимостей (например, "причина -> механизм -> результат").
4. **PASS-4 (Consistency Check):** Финальная TDI-верификация.

## 4. Failure Modes (L1-L2 Interface)
1. **Ambiguity Overload:** Иероглиф имеет > 5 значений в домене. Требуется Axis-9 (Domain Interface).
2. **Structural Drift:** Иероглиф искажен в чертеже (неполная графема). Переход в [VERIFY:image].
3. **Contextual Void:** Отсутствие прагматического маркера в Axis-5. Возврат к Axis-7 (Cognitive).
